# Hello JAX

> **Development Branch Notice**
> This example tracks `main`, which is the NVFlare development branch for the next release.

> that supports a feature, even before that package is published on PyPI.

This example demonstrates how to use NVIDIA FLARE with JAX, Flax, and Optax to train an MNIST classifier with federated averaging (FedAvg). It follows the same hello-world recipe structure as `hello-pt`, while using a JAX client training loop and a flattened parameter vector for model exchange.

## NVIDIA FLARE Installation

For the complete installation instructions, see [Installation](https://nvflare.readthedocs.io/en/main/installation.html).

```bash
pip install nvflare
```

Install the example dependencies:

```bash
pip install -r requirements.txt
```

## Data

This example uses the [MNIST](https://www.tensorflow.org/datasets/catalog/mnist) dataset. The job script downloads the raw MNIST files once before the simulator starts and converts them into `.npy` files. Each client then loads from that prepared cache and partitions the data by site.

## Model

The model is a small convolutional neural network implemented with Flax:

```python
class ConvNet(nn.Module):
    @nn.compact
    def __call__(self, x):
        x = nn.Conv(features=32, kernel_size=(3, 3))(x)
        x = nn.relu(x)
        x = nn.avg_pool(x, window_shape=(2, 2), strides=(2, 2))
        x = nn.Conv(features=64, kernel_size=(3, 3))(x)
        x = nn.relu(x)
        x = nn.avg_pool(x, window_shape=(2, 2), strides=(2, 2))
        x = x.reshape((x.shape[0], -1))
        x = nn.Dense(features=128)(x)
        x = nn.relu(x)
        x = nn.Dense(features=10)(x)
        return x
```

## Client Code

The client script keeps the training loop in JAX while using NVFlare's client API to receive the current global model and send back the updated parameters.

This example flattens the JAX parameter tree into a single NumPy vector before transport, and reconstructs the parameter tree on the client before each round of local training.

## Job Recipe Code

The job recipe uses the base `FedAvgRecipe` configured for NumPy parameter exchange, while the client performs the actual JAX training:

```python
recipe = FedAvgRecipe(
    name="hello-jax",
    min_clients=args.n_clients,
    num_rounds=args.num_rounds,
    initial_ckpt=args.initial_ckpt,
    train_script="client.py",
    framework=FrameworkType.NUMPY,
    server_expected_format=ExchangeFormat.NUMPY,
)
```

Before running the job, prepare two resources:

- an initial flattened parameter checkpoint, generated by `prepare_model.py` and passed to `FedAvgRecipe` through `initial_ckpt`
- the shared MNIST `.npy` cache, prepared once by `prepare_data.py` so both simulated clients do not try to download the dataset at the same time or rely on TensorFlow-only data utilities

## Prepare Assets

Prepare the initial checkpoint and dataset using the default locations under `/tmp/nvflare/data/hello-jax`:

```bash
python prepare_model.py
python prepare_data.py
```

You can also prepare them in custom locations:

```bash
python prepare_model.py --output /path/to/initial_model.npy
python prepare_data.py --data_dir /path/to/mnist
```

## Run Job

After the assets are prepared, run:

```bash
python job.py
```

You can also adjust the default hyperparameters:

```bash
python job.py --n_clients 2 --num_rounds 3 --epochs 1 --batch_size 128
```

If you prepared the assets in non-default locations, pass them explicitly:

```bash
python job.py --initial_ckpt /path/to/initial_model.npy --data_dir /path/to/mnist
```
